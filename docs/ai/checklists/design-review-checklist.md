# Design Review Checklist

- 要件と非ゴールは明示されているか。
- 提案された設計は現在のスコープと整合しているか。
- 重要な用語や責務の境界が明確か。
- なぜその案を選んだのかが説明されているか。
- 必要に応じて、主要な代替案や不採用理由が記録されているか。
- 現段階のプロジェクトに対して十分に単純な設計か。
- 拡張性の前提が、推測ではなく根拠を持っているか。
- データモデル、インターフェース、責務分担が明確か。
- 必要な範囲で異常系や境界条件が考慮されているか。
- 実装やテストに進むための見通しが立つ設計か。
- 設計判断を設計書や ADR までたどれるか。
