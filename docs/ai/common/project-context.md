# Project Context

## 目的

- 釣行の記録をつけることを支援する。
- 過去の記録をもとに、次の釣行計画を立てやすくする。
- 複数エージェントと外部 API を使った開発の練習題材とする。

## プロダクトの方向性

- まずは検証しやすい小さなスコープから始める。
- 記録の質を上げ、その結果として将来の計画精度を高める機能を優先する。
- 人間にも AI にも理解しやすい設計とドキュメント構成を保つ。

## 現在の優先事項

- 実装前に要件を明確にする。
- 設計判断の理由をドキュメントに残す。
- 役割分担型のエージェント運用に適した開発フローを整える。

## 技術スタック

- 言語: Python 3.10+
- 仮想環境・依存管理: uv（利用可能な場合は uv を優先する）
- テストフレームワーク: pytest

## 定義されるまでスコープ外

- 早すぎる最適化
- 具体的な必要性がない複雑なインフラ選定
- ユーザー要求や設計書に裏付けのない機能追加
